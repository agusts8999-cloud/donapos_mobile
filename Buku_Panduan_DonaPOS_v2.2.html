Bab 1: Pengenalan Sistem

DonaPOS adalah sistem Point of Sales (POS) berbasis Cloud yang dirancang dengan arsitektur Hybrid untuk menjamin kecepatan transaksi offline dengan kemudahan manajemen online.

Konsep Backoffice vs Mobile

Komponen Backoffice ERP (Web) Mobile POS (Tablet)
Fungsi Utama Pusat Komando (Manajemen) Ujung Tombak (Operasional)
Pengguna Owner, Manager, Admin Gudang Kasir, Waitress
Kegiatan Input Produk, Ubah Harga, Cek Laporan Input Pesanan, Terima Bayaran, Cetak Struk
Koneksi Wajib Online Bisa Offline (Upload Data belakangan)
Penting: Mobile POS tidak bisa digunakan untuk membuat produk baru. Anda harus membuat produk di Backoffice ERP terlebih dahulu, lalu melakukan Sync Data di Tablet.

Bab 2: Persiapan Awal (Admin)

Langkah ini dilakukan oleh Manajer atau Kapten Shift untuk menyiapkan tablet sebelum digunakan oleh kasir.

2.1 Login Administrator

1. Buka aplikasi DonaPOS Mobile.
2. Di layar Login Utama, masukkan kredensial Admin:
    • Username: tashia
    • Password: 12345
3. Tekan tombol **MASUK**.
4. Sistem akan memverifikasi ke server (membutuhkan internet).

2.2 Sinkronisasi Data (Sync Center)

Setelah login admin berhasil, Anda akan masuk ke layar pemilihan user. Jangan pilih user dulu, tapi lakukan sinkronisasi.

1. Tekan tombol Gerigi (Settings) di kiri bawah untuk masuk ke Admin Dashboard.
2. Pilih menu Sync Center.
3. Tekan tombol SINKRON SEMUA (FULL).
4. Tunggu hingga semua kotak indikator berwarna hijau.
5. Tekan TUTUP / SELESAI.

Bab 3: Operasional Harian (Kasir)

Panduan ini untuk staf kasir (contoh: Aurel) dalam memulai hari kerja.

3.1 Membuka Sesi Kasir

1. Di Layar Pemilihan User, cari kotak bernama AUREL.
2. Klik pada nama Aurel.
3. Layar PIN akan muncul. Masukkan PIN keamanan: 12345.
4. Tekan tombol ENTER (↵).

3.2 Absensi (Clock In)

Sistem DonaPOS mewajibkan absensi sebelum transaksi.

1. Jika belum absen, popup peringatan ATTENDANCE REQUIRED akan muncul.
2. Aplikasi akan membuka kamera depan.
3. Lakukan selfie/foto wajah.
4. Tekan tombol CLOCK IN.
5. Jika berhasil, Anda akan masuk ke layar penjualan (POS Screen).

Bab 4: Manajemen Transaksi

4.1 Alur Transaksi Standar

  1. Tentukan Tipe Pesanan di Header atas (Dine In / Take Away / Online). Harga akan menyesuaikan otomatis.
  2. Pilih Kategori di sebelah kiri (misal: COFFEE).
  3. Tekan gambar produk untuk memasukkan ke keranjang.
  4. Jika produk memiliki varian (misal: Hot/Ice), popup pilihan akan muncul.

4.2 Proses Pembayaran

1. Periksa ringkasan pesanan di panel kanan.
2. Tekan tombol BAYAR SEKARANG.
3. Di Dialog Pembayaran, pilih metode bayar (Tunai, Kartu, QRIS).
4. Untuk Tunai, gunakan tombol Uang Cepat (20k, 50k, 100k) atau ketik manual.
5. Tekan tombol SELESAIKAN TRANSAKSI.
6. Lakukan pencetakan struk (Cetak 1x atau Duplikasi).

4.3 Manajemen Meja

Untuk pesanan Dine In, Anda bisa memilih nomor meja sebelum atau sesudah memilih produk dengan menekan tombol MEJA di header.

Bab 5: Keamanan & OTP (Baru di v2.2)

Untuk melindungi data sensitif (seperti Reset Database atau Pengaturan Server), DonaPOS v2.2 menggunakan sistem keamanan tingkat lanjut.

Mekanisme Challenge-Response

Tidak ada lagi password statis untuk menu Admin tingkat tinggi. Sistem bekerja sebagai berikut:

PENTING: Sistem ini memerlukan 2 perangkat: Tablet Kasir dan HP Vendor (Owner).

1. Di Tablet Kasir

Saat kasir mencoba mengakses menu sensitif, Tablet akan menampilkan 4 Digit Angka Acak (Nomor Respon).
Contoh: 4582

Kasir harus menyebutkan nomor ini ke Owner.

2. Di HP Owner (Aplikasi OTP)

Owner membuka aplikasi DonaPOS OTP Vendor di HP pribadinya. Masukkan angka 4582 tersebut.

Aplikasi HP akan menghasilkan 6 Digit Kode Kunci.
Contoh: 882910

Owner memberikan 6 digit kode tersebut ke Kasir untuk membuka kunci sistem.

Bab 6: Laporan & Closing

6.1 Mencetak Laporan Shift

Dilakukan saat pergantian kasir (misal: Shift Pagi ke Sore).

1. Masuk ke Dashboard Admin (Ikon Gerigi).
2. Pilih menu LAPORAN & CLOSING.
3. Pilih tab Ringkasan Shift.
4. Tekan CETAK LAPORAN SHIFT.

6.2 Closing Harian (Z-Report)

Dilakukan saat toko tutup total.

1. Pastikan semua transaksi hari itu suda di-upload (Menu Posting Sales).
2. Masuk ke menu LAPORAN & CLOSING > tab Z-Report Harian.
3. Periksa total omzet, diskon, dan uang fisik.
4. Tekan tombol merah CLOSING HARIAN.
5. Konfirmasi YA, CLOSING.
6. Struk panjang rekap harian akan tercetak otomatis.
